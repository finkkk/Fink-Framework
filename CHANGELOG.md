# 更新日志

本页面记录 Fink Framework UI 系统的版本更新历史。

---

## 0.1.0  初次提交

发布日期：2025-11-19

**Fink Framework v0.1.0 更新日志**

这是 Fink Framework 的首次公开版本，标志着框架正式进入对外发布阶段。  
该版本提供了一整套面向 Unity 的 模块化开发基础设施，适用于中小型项目快速构建稳定的游戏工程结构。  
核心目标是：降低重复造轮子的成本、提升开发效率、提供统一且可维护的项目架构。   
本次版本包含了框架运行所需的全部核心模块，后续将持续扩展更多工具链与示例内容。

## 0.1.1 

发布日期：2025-11-25

**Fink Framework v0.1.1 更新日志**

本次更新对框架的 数据管线、事件系统、自动绑定工具、场景管理器以及工程结构 进行了大幅增强与结构优化。该版本主要聚焦稳定性、可维护性与模块解耦体验提升。

**1. 事件自动绑定工具：EventAutoBinder**

- 新增一套轻量级的自动注册/解绑方案，大幅减少事件监听的样板代码。
- 新增 Bind()：立即注册事件，OnDestroy 自动解绑
- 新增 BindAuto()：OnEnable 自动注册，OnDisable 自动解绑（适用于 UI 面板等临时对象）
- 完整支持 0 / 1 / 2 参数事件
- 自动避免重复绑定与跨场景残留
- 解决忘记解绑导致的重复回调、内存泄漏等常见问题

**2. 事件系统全面增强**

- 事件模块进行了结构级的升级，确保类型安全、高性能与更多使用场景支持。
- 同一事件名可区分不同参数类型，不再互相覆盖
- 新增 粘性事件（Sticky Event）：新监听者自动接收到最近一次的事件值
- 提升参数匹配效率与全局事件安全性

**3. 数据管线：新增 Excel 公共读取工具**

- 强化数据驱动系统，提升可维护性：
- 新增 ExcelReaderTool，统一表头解析、字段类型检查与错误提示
- DataGenTool / DataExportTool / DataQATool 全部改为使用公共工具读取表头
- 实现 skipColumn 跳列机制，自动跳过不合法或未支持的字段
- 对字段类型引用“未知数据类”的情况新增明确提示（建议先执行“生成数据文件”）
- 数据管线整体结构更一致、更稳定

**4. 工程结构优化**

.gitignore 更新，进一步减少冗余提交

- EditorResManager 移动至 Editor/ 目录（确保仅在编辑器生效）

**5. 数学工具增强（MathsUtil）**

- 新增若干通用数值计算方法，提升工具模块使用广度

**6. 其他维护项**

- 小规模重构部分模块，以提升一致性与可阅读性
- 补充枚举文档注释并优化事件分类结构
- 保持与框架命名规范、目录结构一致

## 0.1.2

发布日期：2025-11-26

**Fink Framework v0.1.2 更新日志**

本次更新主要围绕 音频系统、日志系统、计时器模块、输入系统稳定性 进行全面升级，同时对部分命名与资源加载流程做了结构性优化。

**1. 音频系统重构**
- 引入 **AudioMixer 混音系统**，自动加载主混音器 MasterMixer  
- 新增 Music / SFX 独立混音通道，实现专业级音轨分离  
- 背景音乐改为使用 Mixer 控制音量，移除 AudioSource.volume 直接控制  
- 所有音效（SFX）统一切换至 Mixer Group 路由  
- 全新音量控制逻辑：采用 0~1 → -80~0 dB 映射，更符合专业音频标准  
- `PlaySound()` 行为统一：立即返回 AudioSource，异步加载后自动贴载 Clip  
- 优化异步播放逻辑，提升加载性能与稳定性  

**2. 日志系统增强**
- 新增匿名类的外层类名解析能力，可正确识别 lambda、闭包、协程回调等环境  
- 改进调用者缓存策略，大幅提升日志性能与准确度  

**3. 定时器系统升级**
- 新增 **一次性定时器 SetTimeout**（执行后自动销毁）  
- 新增 **PauseAll / ResumeAll**：一键暂停 / 恢复所有计时器  
- 完善回调销毁与字典清理逻辑，提升稳定性与内存安全性  

**4. 输入系统优化**
- 字典遍历改为 ToArray 快照，避免回调注册/卸载时的遍历异常  
- 使用安全调用 `?.Invoke`，防止空引用错误  
- 调整输入处理结构，提升整体执行效率  

**5. 其他改动**
- 重命名部分对象池脚本，结构更清晰  
- 资源加载管理器新增 **手动清空数据方法**（便于调试与场景切换）
